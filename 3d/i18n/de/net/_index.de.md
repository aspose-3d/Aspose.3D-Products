<p>
 Aspose.3D for .NET ist eine funktions reiche Gameware und computer gestütztes Entwerfen (CAD) API, um Dokumente ohne 3D Modellierungs-und Rendering-Software abhängigkeiten zu bearbeiten. API unterstützt Discreet3DS, WavefrontOBJ, FBX (ASCII, Binär), STL (ASCII, Binär), Universal3D, Collada, glTF, GLB, PLY, DirectX, Google Draco Dateiformate und mehr. Entwickler können den Inhalt von 3D Dokument formaten einfach erstellen, lesen, konvertieren, ändern und steuern.
</p>

<p>
 Aspose.3D for .NET API ist benutzer freundlich und kann mit jeder .NET Anwendung so einfach wie das Kopieren bereit gestellt werden, ohne sich um andere Dienste und Module zu kümmern, die bereits installiert sind.
</p>

   <div class="col-lg-12">
    <h2 class="h2title">
     3D Dateiformat konvertierung
    </h2>
    <p>
     Aspose.3d for .NET bietet eine Reihe von fort geschrittenen Algorithmen, die es zu einer All-in-One-Lösung für die Konvertierung von 3D-Dateiformaten machen. Der Konvertierungs prozess ist so einfach wie das Ändern der Datei erweiterung in das gewünschte Format. Alles, was Sie tun müssen, ist, die Quelldatei in einer Instanz von Scene zu laden und ihre Save-Methode mit einem entsprechenden FileFormat-Parameter aufzurufen. Das war's!
    </p>
    <div class="codeblock" id="code">
     <h3>
      Speichern Sie 3D Szene in verschiedenen Formaten- C#
     </h3>
     <pre><code class="cs">// load the file to be converted

var scn = new Aspose.ThreeD.Scene(dir + "template.fbx");

// save in different formats

scn.Save(dir + "output.stl", Aspose.ThreeD.FileFormat.STLASCII);

scn.Save(dir + "output.obj", Aspose.ThreeD.FileFormat.WavefrontOBJ);</code></pre>
    </div>
   </div>
   <div class="col-lg-12">
    <h2 class="h2title">
     3D Szenen Schöpfung
    </h2>
    <p>
     Aspose.3D for .NET ermöglicht Programmierern, Szenen einfach von Grund auf neu zu erstellen, ohne eine 3D-Modellierungs-oder Rendering software installieren zu müssen. Die 3D .NET API ermöglicht es auch, die Szenen in Formaten (FBX, STL, WavefrontOBJ, Discreet3DS, Universal3D und Collada) zu speichern, indem Sie die Save-Methode des Scene-Klassen objekts aufrufen.
    </p>
   </div>
   <div class="col-lg-12">
    <h2 class="h2title">
     Asset-Informationen zu 3D Szene hinzufügen
    </h2>
    <p>
     Metadaten sind strukturierte Informationen, die das Abrufen, Verwenden und Verwalten einer Informations ressource beschreiben, lokalisieren und erleichtern. Mit Aspose.3D for .NET API können Entwickler Metadaten für die Szene definieren.
    </p>
   </div>
   <div class="col-lg-12">
    <h2 class="h2title">
     Erstellen Sie 3D Netz-und Grafikformen
    </h2>
    <p>
     Mit der .NET 3D Bibliothek können die Entwickler ein Netz mit verschiedenen 3D geometrischen Formen erstellen, Steuer punkte und Polygone auf einfachste Weise definieren, um Maschen zu erstellen und ein Netz auf mehrere Instanzen eines Formtyps zu zeigen. Entwickler können auch 3D Formen an einer beliebigen Stelle in der 3D Szene platzieren.
    </p>

     Teilen Sie Mesh-Geometrie daten mit mehreren Knoten
    </h2>
    <p>
     Um den Speicher bedarf zu verringern, kann eine einzelne Instanz von Mesh an verschiedene Instanzen von Knoten gebunden werden. Stellen Sie sich vor, dass Sie ein System benötigen, in dem eine große Anzahl von 3D Würfeln nicht zu unterscheiden schien. Sie können Speicher sparen, indem Sie ein Mesh-Objekt erstellen, wenn das System beginnt. An diesem Punkt erstellen Sie jedes Mal, wenn Sie eine andere Form benötigen, ein anderes Knoten objekt und zeigen diesen Knoten auf ein Mesh. So verwaltet API Instancing.

Das Speicher layout von Vertex ist auch wichtig, um die GPU (Graphics Processing Unit) zu füttern, da die GPU auch über das Scheitel punkt layout Bescheid wissen muss, um korrekte Attribute aus dem Puffer zu extrahieren.
