        {{< blocks/products/pf/agp/i18n/about-file-text fileFormat="OBJ" readMoreLink="https://docs.fileformat.com/3d/obj/" >}}

OBJ fichiers sont utilisés par l'application Visualiseur avancé de Wavefront pour définir et stocker les objets géométriques. La transmission vers l'arrière et vers l'avant des données géométriques est rendue possible grâce à OBJ fichiers. La géométrie polygonale comme les points, les lignes, les sommets de texture, les faces et la géométrie de forme libre (courbes et surfaces) est prise en charge par le format OBJ. Ce format ne prend pas en charge l'animation ou les informations relatives à la lumière et à la position des scènes. Un fichier OBJ est généralement un produit final du processus de modélisation 3D généré par un CAD (conception assistée par ordinateur). L'ordre par défaut pour stocker les sommets est dans le sens inverse des aiguilles d'une montre, en évitant la déclaration explicite des normales faciales. Bien que OBJ fichiers déclarent des informations d'échelle dans une ligne de commentaire, aucune unité n'a été déclarée pour OBJ coordonnées.


        {{< /blocks/products/pf/agp/i18n/about-file-text >}}

        {{< blocks/products/pf/agp/i18n/about-file-text fileFormat="DAE" readMoreLink="https://docs.fileformat.com/3d/dae/" >}}

Un fichier DAE est un format de fichier Digital Asset Exchange qui est utilisé pour échanger des données entre des applications interactives 3D. Ce format de fichier est basé sur le schéma XML COLLADA (COLLAaborative Design Activity) qui est un schéma XML standard ouvert pour l'échange d'actifs numériques entre les applications de logiciels graphiques. Il a été adopté par l'ISO en tant que spécification accessible au public, ISO/pAS 17506.
